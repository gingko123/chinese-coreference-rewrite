# 进阶实验补充

## 一、为什么增加消融实验

原系统采用规则打分方法判断代词与候选先行词之间的关系。完整分数由距离、位置、类型和性别四类规则信号组成：

```text
final_score = distance_score + position_score + type_score + gender_score
```

为了分析各类规则对系统效果的贡献，本项目增加消融实验。消融实验的基本思想是：在保持其他条件不变的情况下，依次移除某一类规则，观察评价指标变化。

## 二、消融实验设置

实验数据集：

- `test.json`：构造测试集，用于观察规则在标准样本上的稳定性。
- `real_corpus.json`：真实语料风格数据集，用于观察规则在更自然文本中的局限。

实验组：

| 实验组 | 说明 |
|---|---|
| 完整规则 | 使用距离、位置、类型、性别四类规则 |
| 去掉距离规则 | 不考虑候选词与代词之间的距离 |
| 去掉位置规则 | 不考虑候选词是否位于代词之前 |
| 去掉类型规则 | 不考虑人物、物体、组织等类型匹配 |
| 去掉性别规则 | 不考虑“他/她”与姓名性别线索 |
| 仅距离+位置 | 只保留浅层位置类信号 |
| 仅类型+性别 | 只保留实体属性类信号 |

## 三、实验结果生成方式

运行：

```bash
python scripts/run_ablation.py
```

会生成：

```text
reports/ablation_results.md
```

Streamlit 页面中也新增了“消融实验”标签页，可以直接选择数据集并查看实验结果。

## 四、结果分析

从实验设计上看，完整规则通常表现最好，因为它同时考虑了局部距离、先后位置、实体类型和人物性别。对于构造测试集，样本模式较规范，规则组合可以稳定处理大部分指代关系。

在真实语料风格数据集中，规则方法会暴露更多问题。例如：

```text
上海市启动了城市更新计划，该城市希望它改善居民生活环境。
```

该样本中，系统容易受到距离规则影响，把“该城市”错误指向更近的“计划”，或者把“它”错误指向“城市”。这说明单纯依赖距离和类型规则，很难处理需要语义理解的复杂样本。

## 五、BERT 语义匹配增强方案

后续可以加入 BERT 或 sentence-transformers 作为语义匹配模块。基本思路如下：

1. 对每个代词构造上下文窗口。
2. 对每个候选先行词构造上下文窗口。
3. 使用中文 BERT / MacBERT / sentence-transformers 编码两段文本。
4. 计算上下文向量相似度，得到 `semantic_score`。
5. 与原规则分数融合：

```text
final_score = 0.7 * rule_score + 0.3 * semantic_score
```

这种方法有两个优点：

- 能缓解多个候选实体距离相近时的歧义问题。
- 能利用上下文语义判断“它”“该城市”等表达更可能指向哪个实体。

## 六、面向问答场景的应用

指代消解结果可以服务问答系统。例如用户提出问题：

```text
小明把书给了小红，因为她需要它。谁需要书？
```

如果系统先完成指代消解和改写：

```text
小明把书给了小红，因为小红需要书。
```

那么问答模块可以更容易得到答案：

```text
小红。
```

因此，本项目不仅可以作为独立的文本改写工具，也可以作为问答、摘要、信息抽取等下游任务的预处理模块。

## 七、进阶工作的意义

加入消融实验后，报告不再只是描述“系统能运行”，而是能够回答：

- 哪些规则真正有用？
- 系统在哪类样本上容易出错？
- 真实语料和构造语料有什么差异？
- 为什么需要进一步接入 BERT 或 HanLP/LTP？

这能显著增强项目的实验完整性和课程报告说服力。
